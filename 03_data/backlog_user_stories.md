Backlog User Stories — tracking products tool 
Edgard&Cooper | Version 1 | 23/03/2026

| ID | User Story | Priorité MoSCoW | Critère d'acceptation |
|---|---|---|---|

| US-01 | En tant que Directeur e-commerce, je veux recevoir une alerte Slack le lundi matin listant les produits OOS par marché afin de réagir rapidement | Must Have | L'alerte est envoyée chaque lundi avant 9h avec la liste des ASINs OOS par marché |

| US-02 | En tant que directrice marketing, je veux m'assurer que le budget marketing n'est pas utilisé pour un produit invendable. | Must Have | Un rapport hebdomadaire OOS est accessible à la Directrice Marketing chaque lundi |

| US-03a | En tant que responsable e-commerce, je veux connaître plus vite possible les produits OOS, leur durée d'OOS | Must Have | Alerte est envoyée chaque lundi avant 9h avec les produits OOS |

| US-03b | En tant que responsable e-commerce, je veux savoir si les prix sont corrélés au listing price. | Must Have | Alerte est envoyée chaque lundi et jeudi avant 9h avec les produits ayant connu de forte baisse ainsi que les produits avec un prix au delà d'un certain % |

| US-04 | En tant que membre de l'équipe Supply Chain, je veux avoir une vue sur des produits OOS s'ils ne l'ont pas remarqués avant leur permettant de réagir plus rapidement | Must Have | Alerte est envoyée chaque lundi avant 9h avec la liste des ASINs OOS par marché |

| US-05 | En tant que Country Manager d'un pays, je veux recevoir un rapport mensuel sur les chiffres du e-commerce avec le nombre moyen de produits OOS + les % de variations de prix des catégories de produits. | Could have | Report en data viz sur le nombre de produits OOS moyen sur le mois / Baisse moyenne de toutes les catégories de produits pour voir si il n'y a pas quelque chose à changer |
